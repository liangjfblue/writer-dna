## 开头模式

卡兹克常用的 4 种开头方式，每种都从一个具体的、当下的切入点进入：

### 1. 事件/动作切入

从一个正在发生的具体动作或事件开始，让读者直接"走进"场景。

> 示例：凌晨三点，我偷偷起床，打开电脑，开始跑一个我完全看不懂的脚本。

要点：有时间、有地点、有动作、有情绪。读者一秒钟就能代入。

### 2. 悬念/反常切入

抛出一个反直觉或让人好奇的事实，制造"然后呢？"的冲动。

> 示例：我在淘宝上花 9.9 买了个 DeepSeek，结果它教我做的事，比大多数付费课程都离谱。

要点：反常 + 具体结果。不要只说"有个东西很有意思"，要说"它做了什么"。

### 3. 自嘲/私人切入

从自己的窘境、无知或小失误切入，卸掉读者的防御心。

> 示例：说实话，我第一次看到这个工具的时候，完全没看懂它在干嘛。愚钝如我，折腾了整整两个小时。

要点：真实的不完美 > 虚假的完美。越具体的窘境越有代入感。

### 4. 数据/现象震撼切入

用一个让人"卧槽"的数据或现象开头，直接拉满好奇心。

> 示例：一个卖 9.9 的产品，24 小时内 1000 人付款。你以为是个笑话？人家的收入可能比你月薪还高。

要点：数据要有冲击力，紧接着用"你以为...其实..."反转。

## 逐一展示法（升番逻辑）

当文章涉及多个产品/案例对比时，不要一次性罗列，而是逐一展示，每次展示都比上一次更震撼——这就是"升番"。

### 升番结构

```
第 1 个：还行，有点意思（建立基准线）
第 2 个：等等，这个有点东西（超出预期）
第 3 个：卧槽？？？（彻底震撼）
```

### 写作要点

- 每个产品/案例用一段完整的故事包裹，不是干巴巴的参数对比
- 每个之间要有"情绪转场"，比如"我以为这就到头了，直到..."
- 最后一个必须是全场最佳，让读者觉得"值了"
- 适合场景：产品横评、工具推荐、多案例展示

### 示例节奏

> 第一个工具，说实话，一般般。能跑，但也没什么特别的。
>
> 然后我试了第二个。嗯？这个有点意思啊。
>
> 等你看到第三个的时候，你就知道我为什么写这篇文章了。

## 创意案例微型故事模板

产品评测或工具推荐时，不要说"这个工具可以帮你做 X"，而是把功能包装成一个微型故事，让读者自己感受到"卧槽"。

### 故事结构（3-5 句话）

```
触发（具体场景 + 具体人物）→ 行动（使用工具/方法）→ 结果（震撼/反常/感动）
```

### 写作要点

- 人物必须具体：不是"一个用户"，而是"一个四五线城市刚毕业的大学生"
- 场景必须真实：不是"在某个场景下"，而是"凌晨两点在十几平米的出租屋里"
- 结果必须有情绪：不是"提高了效率"，而是"他当时就愣住了"
- 适用场景：工具分享型、产品体验型文章

### 反面 vs 正面

反面："这个工具可以帮助用户节省大量时间，提升工作效率。"

正面："我朋友小王，做设计的，以前改一版图要两个小时。用了这个之后，十五分钟搞定。他跟我说的时候表情跟见了鬼一样。"

## 结构模板

### 通用文章结构

```
开头（感性切入，100-200 字）
  → 让读者产生"然后呢？"的冲动

背景/科普（300-500 字，可省略）
  → 用最短的时间让读者理解背景
  → 知识是"聊着聊着顺手掏出来"的

核心内容（分板块展开，占全文 60-70%）
  → 每个板块：观点 + 支撑（案例/数据/个人经历）+ 私人视角 + 扣主线句
  → 板块之间用口语化转场衔接（"说到这个""回到 xxx 这块"）
  → 适合长文时用一句话独立成段制造"断裂"效果

升华（200-400 字）
  → 从具体事件连接到更大的文化/哲学/历史参照物
  → 不是硬凑的，是"聊着聊着自然想到了"

收尾（100-200 字）
  → 回扣开头（契诃夫之枪）
  → 或用一句有力量的话结尾
  → 或留白让读者自己想
```

### 调查实验型结构

```
好奇/动机 → 行动过程（踩坑、意外、发现）→ 结果展示（卧槽时刻）→ 升维思考
```

### 产品体验型结构

```
为什么关注 → 场景演示（真实使用过程）→ 效果/感受 → 与其他产品的自然对比 → 总结
```

### 方法论分享型结构

```
谦逊铺垫（"不成熟的经验"）→ 方法 1（观点 + 案例 + 可执行行动 + 坦诚失败点）
→ 方法 2 → ... → 回扣所有行动点 + 升华
```

## 收尾方式

### 1. 回环呼应

前面埋的钩子在结尾 callback。比如开头提到"磨平一些信息差"，结尾回扣"哪怕只磨平了一点点"。

### 2. 哲思留白

用一句开放式的话结尾，让读者自己想。

> 示例：大时代啊，朋友们。

### 3. 行动呼吁（克制版）

不是"点赞关注"，而是跟文章主题相关的行动。

> 示例：你也可以回想一下，上一次你对一个新事物感到兴奋，是什么时候？
