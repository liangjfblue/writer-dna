03 — Concepts

三个概念的融合

WriterDNA 是 Karpathy 式极简知识库、卡兹克式 Skill 架构、和蒸馏范式的结合体。
● 知识库

Karpathy 式知识库

像 Karpathy 把 LLM 知识拆解成结构化、可理解、可学习的模块一样，WriterDNA 把写作风格拆解成可查询、可迭代的知识库。

人格轨：核心价值观、语气节奏
技法轨：词库、技法、自检规则
● Skill

卡兹克式输出

最终产出是一个符合 Agent Skills 开放标准的写作 Skill，包含风格内核 + 五层自检体系。

SKILL.md + references/ + scenes/
安装即用，对 Claude 说"帮我写"
● 蒸馏

Distill 范式

从原始素材中"蒸馏"出风格基因，注入骨架模板。写作中持续纠错，风格越来越像你。

情况 A：提取自己的风格
情况 B：学习目标作者的风格
04 — Why

为什么不是直接写个 Prompt

五个让 WriterDNA 不只是"又一套写作模板"的设计决策。
⬡

双层架构

骨架模板（写作框架）和风格数据（你的风格基因）分离，两层独立迭代。换骨架不需要重新分析风格，更新风格不需要改骨架。

⊞

插件化场景

每种文章原型是独立插件，包含专属的开头模式、结构模板和收尾方式。新增类型零侵入，核心 SKILL.md 不动。

◎

五层渐进式自检

L1 硬规则（零容忍）→ L2 风格 → L3 质量 → L4 活人感 → L5 运行时纠错。L5 是杀手级功能：写作中说"这句不像我"，自动记录，优先级高于 L1。

⇄

双用户类型适配

有经验写作者提取自己的风格（[CONFIRMED]），纯新手从喜欢的作者学习（[LEARNED]），同一套系统，随用户成长自动过渡。

◈

可追溯的风格知识库

wiki/ 不是黑盒。每个技法有证据标签，补充素材后增量更新，随时可审可调。

04 — Structure

项目结构

干净的四层架构：规则 → 模板 → 素材 → 产出。
writer-dna/ ├── CLAUDE.md ← 蒸馏引擎（AI 指令） ├── templates/ ← 骨架模板（风格无关的写作框架） │ └── khazix-writer/ │ ├── SKILL.md │ └── references/ ├── raw/ ← 你的原始素材（AI 只读不改） │ ├── profile.md │ ├── articles/ │ ├── chats/ │ ├── references/ │ └── interviews/ ├── wiki/ ← 风格知识库（AI 分析生成） │ ├── persona/ ← 人格轨 │ └── craft/ ← 技法轨 └── outputs/ ← 生成的 Skill 包 └── <your-name>-writer/ ├── SKILL.md └── references/ ├── style_examples.md ├── content_methodology.md └── scenes/
